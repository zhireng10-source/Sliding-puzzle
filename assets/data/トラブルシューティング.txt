====================================
リアクション吹き出しのトラブルシューティング
====================================

【問題】同じセリフばかり表示される

【原因と対処法】

1️⃣ ブラウザキャッシュの問題
  ブラウザが古いJSONファイルをキャッシュしている可能性があります。

  【対処法】
  - Ctrl + F5 でページを強制再読み込み
  - または、Ctrl + Shift + Delete でブラウザのキャッシュをクリア

2️⃣ JSONファイルの読み込み確認
  JSONファイルが正しく読み込まれているか確認してください。

  【確認方法】
  1. ゲームを開く
  2. F12キー を押して開発者ツールを開く
  3. 「Console」（コンソール）タブを選択
  4. ボーナスステージを開始する
  5. 以下のログが表示されるか確認：

     📖 リアクションテキスト読み込み開始...
     ✅ REACTIONS_DATAが見つかりました
     ✅ リアクションテキストを読み込みました:
       - 通常リアクション: 249 個
       - ダブルタップ: 15 個
       - 連打: 15 個

  【エラーが表示される場合】
  - ❌ REACTIONS_DATAが見つかりません:
    → reactions-data.jsが読み込まれていません
    → index.htmlで正しく読み込まれているか確認してください
      正しい場所: js/reactions-data.js
    → HTMLファイルで以下の行があるか確認：
      <script src="js/reactions-data.js"></script>

3️⃣ クリック時のリアクション確認
  クリックした時にどのリアクションが選ばれているか確認してください。

  【確認方法】
  1. F12キー → Console タブ
  2. CGをクリックする
  3. 以下のようなログが表示されます：

     🎭 リアクション表示 [42/252]: 「○○○…」

  同じ番号ばかり表示される場合は、乱数生成に問題があります。
  毎回違う番号が表示されるのに同じセリフが表示される場合は、
  JSONファイルに同じセリフが複数含まれている可能性があります。

4️⃣ JavaScriptファイルの構文エラー
  reactions-data.jsを編集した後、構文エラーがある可能性があります。

  【確認方法】
  1. reactions-data.jsをテキストエディタで開く
  2. 以下を確認：
     - 全てのセリフが "「」" で囲まれているか
     - セリフの間に , （カンマ）があるか
     - 最後のセリフの後にはカンマがないか
     - { } [ ] の対応が正しいか
     - window.REACTIONS_DATA = で始まっているか
     - 最後に ; （セミコロン）があるか

  【修復方法】
  - バックアップがあれば復元
  - なければ、reactions-data.jsを削除して再作成

5️⃣ スクリプト読み込み順序の問題
  reactions-data.jsがclicker-game.jsより後に読み込まれている場合、
  REACTIONS_DATAが使えません。

  【対処法】
  index.htmlで以下の順序になっているか確認：
  1. <script src="js/reactions-data.js"></script>
  2. <script src="js/clicker-game.js"></script>

【まだ解決しない場合】
  開発者ツールのConsoleタブのログをすべてコピーして、
  エラーメッセージを確認してください。
